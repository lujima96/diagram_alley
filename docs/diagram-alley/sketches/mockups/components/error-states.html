Error States

F5 §9.1 — Error banners and full-panel error states for key failure modes. Shown as isolated component states.

SPEC_VALIDATION_FAILED — inline banner in workspace preview panel
F5 §9.1 · Shown when: AI generation returns an invalid spec that cannot be auto-repaired
SPEC_VALIDATION_FAILED · Generation returned an invalid spec
Diagram spec is invalid
The AI returned a spec with 2 errors that could not be auto-repaired:
  • Edge "api → auth_service" references missing node "auth_service"
  • Node "postgres_db" ID is duplicated
Your previous diagram is unchanged. Use the Validation tab to fix errors manually.
PROVIDER_UNAVAILABLE — inline banner during generation
F5 §9.1 · Shown when: AI provider returns 502 or connection timeout
PROVIDER_UNAVAILABLE · GPT-4o (BYOK) could not be reached
AI provider unavailable
GPT-4o (BYOK) did not respond. Check your API key and endpoint, or switch to a different provider.
Your diagram was not changed. The workspace remains fully functional for manual editing.
SUBSCRIPTION_REQUIRED — upgrade banner (post-trial access gate)
F5 §9.1 · D7 §2.3 · Shown when: trial expired and user tries to save or use AI
SUBSCRIPTION_REQUIRED · Trial has expired
Your trial has expired
Saving and AI generation require an active Pro subscription. Your diagrams are safe and can still be viewed and exported.
RATE_LIMITED — inline banner (429 from provider or app)
F5 §9.1 · Shown when: too many AI generation requests in a short window
RATE_LIMITED · Too many requests
Too many requests
Your AI provider returned a rate limit error. Please wait a moment before generating again.
or edit manually while waiting

D1 §5 · Shown when: user opens workspace and no AI provider is set up
No AI provider configured
Connect an Ollama endpoint or add a cloud API key to use AI generation. Manual editing is always available.
