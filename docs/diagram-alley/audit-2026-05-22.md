## 2. Decisions Resolved During Reconciliation

### 2.1 Project Archive / Restore UX

**Original gap:** F1 defines `projects.is_archived`, but no spec defined whether archived projects are viewable, restorable, or hard-deletable.

**Resolution:** Closed by DEC-027. V1 supports project archive and restore, shows archived projects in an Archived filter/view, and does not expose project hard delete as a standalone action.

**Updated owners:** F1, F4, F5.

---

### 2.2 Diagram Move Between Projects

**Original gap:** D2 allows `project_id` updates in `PATCH /api/v1/diagrams/{id}`, but no UX or product decision explicitly said whether users can move diagrams between projects.

**Resolution:** Closed by DEC-028. Users can move diagrams between active projects through a diagram menu action using the existing `PATCH project_id` field.

**Updated owners:** D2, F4.

---

### 2.3 User Display-Name Fallback

**Original gap:** F1 allows `users.name` to be null. D6 returned `shared_by_name = null` if no name existed. Mockups display initials and names.

**Resolution:** Closed by DEC-029. The fallback display name is the lowercased email prefix before `@`; avatar initials derive from the display name or email prefix.

**Updated owners:** F1, D6.

---

### 2.4 Visual Editor Appearance for `file_structure` and `ui_wireframe`

**Original gap:** F2/F4 defined React Flow node appearance for architecture-style nodes but did not define how file structure nodes or UI wireframe components render in the visual grid editor.

**Resolution:** Closed by DEC-030. File structures render as editable tree rows; UI wireframes render as editable nested boxes.

**Updated owners:** F2, F4.
